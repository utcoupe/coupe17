Definition at line 233 of file Helpers.h.

#define OS48_VOLATILE_R (   TYPE,
  VARIABLE 
)

This macro ensures you read the value of the variable from the memory.
Because the compiler can re-order and optimize some operations, it can store the first time the value in a register and then always read the value from this register. In a concurrent tasks context, by using the same variable, the first task having stored the value in a register doesn't "see" the modification from the second task that stores the new value in memory. OS48_VOLATILE_R allows you to read the value of the variable from memory to avoid this case.

Parameters
TYPEThe type of the variable
VARIABLEThe variable to read
Returns
The value

Example : Suppose, for some reasons of optimizations, you don't want to set val as a volatile variable. But in the following case you have to read the variable from the memory.

byte val = 0;
void function task1()
{
while (OS48_VOLATILE_R(byte, val) == 0) {}
[...]
}
void function task2()
{
os48_delay(1000);
OS48_VOLATILE_R(byte, val) = 1;
}
Note
You don't have to use this macro inside a critical section because it prevents already the problems.
